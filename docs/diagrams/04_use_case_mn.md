## Тайлбар

Use case диаграм нь системийн **гадаад үйл ажиллагаанаас** харсан функцыг харуулдаг — техникийн нарийн дотоод бүтцээс (классууд, баазын схем) тусдаа. Boloroo системд **3 actor**:

1. **Энгийн хэрэглэгч** — чат хийдэг, ангилал сонгодог, санал өгдөг.
2. **Админ хэрэглэгч** — баримт оруулдаг, статистик харах, систем хяналт хийдэг. *(Эрсдэл: одоогоор admin endpoint-д нэвтрэлт байхгүй учир эдгээрийг хэн ч хийж болно — хамгаалалтын өмнө сайжруулах ёстой.)*
3. **Систем** — backend дотор автоматжсан үйл явц: classification, retrieval, generation, logging.

Use case-ууд нь дотроо **«include» хамаарал** үүсгэдэг: жишээ нь, *«Асуулт асуух»* нь *«Аюулгүй байдлын ангилалт»*, *«Мэдлэгийн санд хайлт»*, *«LLM-р хариулт»* гурвыг хамтдаа гүйцэтгэх ёстой.

## Дипломын тайланд ашиглах тайлбар

Use case диаграм нь дипломын ажлын **«Системийн шаардлагын анализ»** бүлэгт чухал. Уг диаграм нь:

- **Functional requirements** (юу хийх ёстой)-г хүний-уншихуйц хэлбэрээр харуулна.
- **Actor-based use case-ууд** нь хэрэглэгчийн төрөл бүрийн зорилгод чиглэсэн featurнүүд байгаа эсэхийг харуулдаг.
- **Include-relations** нь системийн дотоод coupling-ыг (нэг үйл явц өөр үйл явцыг шаардаж буй эсэх)-ыг тусгана.

Энэ диаграм нь **багшийн «системийн зорилго юу вэ?»** гэсэн анхны асуултад шууд хариу болно. Boloroo бол **боловсрол + хууль зүйн зөвлөгөө + аюулгүй байдлын шүүлт** гурвыг нэгтгэсэн чатбот гэдэг нь use case-аас тодорч харагдана.

Гол анхааруулга: одоогоор **role-based access control** хэрэгжээгүй. «Энгийн хэрэглэгч» болон «Админ хэрэглэгч» хоёрыг ялгахын тулд UI-тай харагдах төгсгөлийн товчлуурыг харуулдаг ч backend-д ямар ч баталгаажуулалт байхгүй. Энэ нь **диплом-after** ажлын жагсаалт болж орох ёстой (FIX_PLAN_MN.md-д бичсэн).

## Хамгаалалтын үеэр тайлбарлах богино хувилбар

«Системд 3 актор: Энгийн хэрэглэгч асуулт асууж хариу болон эх сурвалж авна, санал өгнө. Админ хэрэглэгч баримт нэмэх, устгах, статистик харах боломжтой. Систем өөрөө аюулгүй байдлын шалгалт, retrieval, LLM хариу үүсгэх, лог хадгалах гэх мэт автомат үйл явц гүйцэтгэдэг. Use case-ууд нь дотроо include хамаарлаар холбогдсон — нэг асуулт асуухад 3 дотоод үйл явц цогцоор гүйцэтгэгддэг.»
